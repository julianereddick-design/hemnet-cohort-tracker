## HARD GUARDRAILS — read before running anything (from 22-CONTEXT.md, NON-NEGOTIABLE)

This is a **LIVE, team-owned production droplet running a live scraper.** Operate READ-ONLY.

**FORBIDDEN (do not run any of these, in any form):**
- Any write or file edit on the droplet; `rm`, `mv`, `truncate`, `>`/`>>` redirection into droplet files, log truncation
- `systemctl stop|restart|disable`, killing any process, service restart/stop
- ANY docker mutation: `docker restart|stop|kill|rm|prune|pull|build|exec <write>`, volume/image removal
- Package install (`apt`, `pip install`, `npm install`), config change, crontab edit
- `reboot`, `shutdown`, or anything that changes droplet state

**ALLOWED (inspection only):** `ls`, `cat`, `head`, `tail`, `wc -l`, `stat`, `du`, `df`, `free`, `ps`, `top -bn1`, `uptime`, `nproc`, `systemctl status`, `docker ps`/`docker inspect`/`docker logs --tail`/`docker stats --no-stream`/`docker volume ls`/`docker system df`, `crontab -l`, `redis-cli INFO`/`DBSIZE`/`--scan`, **read-only** `psql` (`\l`, `\dt`, `SELECT ... count/size`), `celery ... inspect`, `grep`/`find` over small source/config dirs.

**BOUND HEAVY I/O:** never `du -sh /` whole-tree; never grep/cat the 4.6 GB `kill.log` or the 121 MB blocking log — get their size with `ls -la`/`stat`/`du -sh <file>` only. Use `du -sh <specific dir>` per top-level dir, `find ... -maxdepth N`, and `head`/`tail`/`wc -l` instead of full reads.

**SECRETS:** if a settings/env file exposes a DB password, API token, or key, record only its **location and the fact it exists** in the evidence file — NEVER paste the secret value. Redact as `<REDACTED>`.

## Connection (exact — from the runbook; note the IdentitiesOnly gotcha)
Run remote commands through the Bash tool (Git Bash; `~` resolves to your home). Define the prefix once per task:
```bash
DROPLET='ssh -o IdentitiesOnly=yes -o IdentityAgent=none -o ControlMaster=no -o ControlPath=none -o ConnectTimeout=12 -i ~/.ssh/droplet_ed25519 root@170.64.181.89'
$DROPLET 'docker ps -a'
```
Without `-o IdentitiesOnly=yes -o IdentityAgent=none` the agent offers other keys first, hits `MaxAuthTries`, and you get a FALSE `Permission denied (publickey)` (see runbook). Always pass the full flag set.
