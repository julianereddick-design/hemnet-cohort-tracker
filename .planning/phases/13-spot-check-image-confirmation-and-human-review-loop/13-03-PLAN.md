<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo → Slack API | A workspace-level bot token (xoxb-) authenticates outbound posts + inbound reaction reads. Token compromise = ability to post as the bot and read channel reactions. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-08 | Information Disclosure | SLACK_BOT_TOKEN | mitigate | Token read only from `process.env.SLACK_BOT_TOKEN`, never hardcoded, never logged (the lib logs failures without echoing the token), `.env` is git-ignored; runbook explicitly states "never commit / never log". |
| T-13-09 | Spoofing | Authorization header | mitigate | Every API call sends `Authorization: Bearer <token>`; least-privilege scopes (chat:write + reactions:read only — no admin, no channels:write). |
| T-13-10 | Denial of Service | slack API request | mitigate | 10s timeout + destroy on every request; any error returns null so a Slack outage degrades (no review post) rather than crashing the gate/poller. |
</threat_model>
