<objective>
Implement **Mode B** — the automated Claude-vision adjudicator — as a cleanly separable layer
on top of the Mode A pipeline from Plan 12-02. Add `lib/spotcheck-vision.js` (calls the
Anthropic API with the downloaded gallery images to decide whether two listings show the same
physical property), install `@anthropic-ai/sdk`, add the `ANTHROPIC_API_KEY` env var, and wire
the gate's `--mode-b` flag to call vision ONLY for the pairs the deterministic triage flags
(`suspect` + `low-signal`) — never for already-resolved `likely-match` pairs (cost gate).

Mode B is strictly additive: when `--mode-b` is off OR `ANTHROPIC_API_KEY` is unset, the gate
behaves exactly as Plan 12-02 (Mode A). This keeps the API integration deferrable and the
deterministic artifact complete on its own, per the CONTEXT.md KEY FORK.

Purpose: make the weekly run fully hands-off by letting Claude render the per-pair "same
property?" judgment the photo layer needs.
Output: `lib/spotcheck-vision.js`, gate wiring, `@anthropic-ai/sdk` dep, `.env.example` entry.
</objective>

adjudicatePair expects, per pair, an optional:
```javascript

```
Pair input to vision (from the artifact after spotcheck-photos.js --gallery):
```javascript

```
adjudicatePairs accepts:  adjudicatePairs(records, { visionResults })  where
  visionResults is a map { [pair_id]: { sharedPhoto, confidence, reasoning } }.

Anthropic SDK (NEW dependency — no analog in repo). Node usage shape:
```javascript
const Anthropic = require('@anthropic-ai/sdk');
const client = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });
const msg = await client.messages.create({
  model: '<latest vision-capable Claude model>',   // see note below
  max_tokens: 512,
  messages: [{ role: 'user', content: [
    { type: 'text', text: '<prompt>' },
    { type: 'image', source: { type: 'base64', media_type: 'image/jpeg', data: '<b64>' } },
    ...
  ]}],
});
// msg.content[0].text holds the model's reply
```
Model note: use the latest vision-capable Claude model id. Read it from an env override
`ANTHROPIC_MODEL` if set, else default to a current model string (e.g. the latest
`claude-...` released model). Do not hardcode a deprecated id.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gate → Anthropic API | Property images + listing context (addresses via filenames are NOT sent; only image bytes + a generic prompt) cross to a third-party API — DATA EGRESS. |
| local disk → vision module | Downloaded gallery JPEGs are read and base64-encoded for the request. |
| env → SDK | ANTHROPIC_API_KEY is read from process.env only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-09 | Information disclosure | ANTHROPIC_API_KEY secret | mitigate | Key read ONLY from `process.env.ANTHROPIC_API_KEY` inside getClient(); never hardcoded; .env.example carries a placeholder only; real key lives in gitignored .env. Grep-asserted no literal key. |
| T-12-10 | Information disclosure | property images + addresses → Anthropic (data egress) | accept (documented) | Egress is the intended function of Mode B (vision judgment). Images are public listing photos. Mitigations: send image bytes + a generic same-property prompt only (no PII beyond what the public listing shows); gate behind triage so only suspect/low-signal pairs (a small subset) are sent; Mode B is opt-in (--mode-b + key). Documented as an accepted egress in the runbook. |
| T-12-11 | Denial of Service / cost | per-run vision call volume | mitigate | Vision is called ONLY for suspect+low-signal pairs (typically a handful), capped at maxImagesPerSide images each; likely-match pairs never hit the API. Bounds both latency and spend. |
| T-12-12 | Tampering | malformed model JSON reply | mitigate | Parse defensively; any parse/API error → return null → caller treats the pair as Mode A (UNCERTAIN), never crashes the gate. |
| T-12-13 | Spoofing | image files read from disk | mitigate | Only files under the gate-created artifactDir (written by our own spotcheck-photos.js) are read; paths come from our artifact JSON, not user input. |
</threat_model>
