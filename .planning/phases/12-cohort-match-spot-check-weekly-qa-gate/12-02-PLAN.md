<objective>
Build `cohort-spotcheck-gate.js` — the weekly orchestration entrypoint that wraps the
ALREADY-BUILT spot-check tools in `cron-wrapper.runJob`, runs the full deterministic pipeline
(resolve latest cohort → sample + field evidence → photo galleries → adjudicate → summary),
writes the verdicts + summary artifacts, logs to `cron_job_log`, and escalates via Slack when
the confirmed false-match rate exceeds the threshold (default > 5%) or on fetch failure.

This plan delivers **Mode A only** (deterministic + human-fallback adjudication, NO Anthropic
API). The pipeline is mode-agnostic: it calls `adjudicatePairs` with no vision results, so the
artifact is complete and usable today. Mode B (Claude vision) plugs in cleanly in Plan 12-03.

Purpose: a hands-off weekly quality gate that measures the false-match rate and alerts on it.
Output: `cohort-spotcheck-gate.js`, a crontab/runbook entry, VERDICTS + SUMMARY artifacts.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| gate → child tools (cohort-spotcheck.js / spotcheck-photos.js) | The gate spawns Node child processes with cohortId resolved from the DB. |
| child tools → Hemnet/Booli (live HTML + image CDNs) | Untrusted live HTML/images are fetched by the EXISTING tools (already hardened in prior phases). |
| gate → Slack | Escalation text crosses to Slack via cron-wrapper's existing webhook. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-04 | Tampering / Injection | execFileSync argv (cohortId) | mitigate | Use `execFileSync('node', [args])` (argv array, NOT a shell string) so cohortId can never be shell-interpreted. cohortId originates from our own `cohorts` table, not user input. |
| T-12-05 | Information disclosure | SLACK_WEBHOOK_URL secret | mitigate | Never read or hardcode the webhook in the gate; escalation flows solely through cron-wrapper which reads `process.env.SLACK_WEBHOOK_URL`. Grep-asserted no hardcoded webhook. |
| T-12-06 | Spoofing / live HTML | Hemnet/Booli pages + images fetched by child tools | accept (inherited) | Fetching/parsing is delegated to the already-hardened cohort-spotcheck.js / spotcheck-photos.js (direct→Oxylabs scrape layer); no new fetch path introduced here. |
| T-12-07 | Repudiation | run accountability | mitigate | Every run writes a `cron_job_log` row (started/finished/status/result_summary) via runJob — full audit trail. |
| T-12-08 | Denial of Service | child-process runtime / Oxylabs cost | mitigate | `--rate` (8% sample) + `--max` gallery cap bound fetch volume; cron-wrapper's 120s statement_timeout + JOB budget conventions apply; sample is ~110 pairs. |
</threat_model>
