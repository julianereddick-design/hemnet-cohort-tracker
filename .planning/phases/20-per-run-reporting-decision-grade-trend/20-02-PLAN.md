Build `sold-match-trend-chart.js` — a standalone committed-HTML Chart.js-4 trend generator over
the `sold_match` table. For each fortnightly period (keyed by `window_end` ISO week) it computes
the **settled genuine-non-Hemnet rate** and the **match rate**, and writes a self-contained HTML
chart to `view-data/<date>/sold-match/trend.html` (served by the existing `view-data-server.js`
on :3800).

series (D-03/D-07) so slutpris-lag contamination is never read as genuine non-Hemnet presence.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-20-06 | Tampering (XSS) | Embedded JSON in <script> | mitigate | Only numeric rates + ISO-week-key strings (regex `\d{4}-W\d{2}`) are embedded via JSON.stringify; no free-text/address/PII reaches the HTML, so no script-injection surface |
| T-20-07 | Information disclosure | view-data/ static file | accept | view-data-server already serves this dir (existing surface); the chart carries only aggregate rates, no per-property data — no new exposure |
| T-20-08 | Tampering | Unparseable segment string | mitigate | Per-region derivation buckets unknowns under 'Unknown' + WARN; national line never depends on segment parsing, so a bad segment cannot break the mandatory output |
