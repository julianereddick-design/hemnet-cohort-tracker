<objective>
Build `sold-match-report.js` — a standalone per-run Slack summary over the `sold_match`
table. It buckets each row's verdict (matched / booli_only-in-recheck / re-check-resolved-late
/ settled-non-Hemnet / uncertain) per segment, rolls up to region and national, and posts a
monospace Slack block via `lib/spotcheck-slack-bot.js`.

THE headline is the **settled genuine-non-Hemnet rate** = `genuine_non_hemnet / (matched +
genuine_non_hemnet)` over terminal verdicts only (D-03), reported DISTINCTLY from the raw
`booli_only` rate (D-04) so slutpris-lag contamination is never read as genuine non-Hemnet
presence.

Purpose: REPORT-01 (per-run summary) + REPORT-03 (decision-grade settled headline).
Output: `sold-match-report.js` with a `--smoke` offline acceptance gate.
</objective>
