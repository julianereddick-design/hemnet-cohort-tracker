<objective>
Build `lib/sold-sample.js` — the national population-weighted fortnightly sampler (D-13/D-14).
It reads `config/sold-panel.json`, fetches each panel muni's Booli `/slutpriser` 14-day feed
per object type (Hus, Lägenhet), excludes deeds and de-dupes against the `booli_sold` rows
already stored, then ALLOCATES the ~1,000 target across munis by population (capped at live
14-day volume) with the natural within-muni Hus:Lägenhet ratio, and TAGS each sampled record
with a synthetic per-record `seg` so the existing `matchOne` can match it. The allocation math
is a PURE function (offline-unit-testable); the Booli fetch + de-dup query are injectable for
the offline `--smoke`.

Purpose: SCHED-01 — this sampler IS the seeding layer the Plan 19-02 orchestrator drives (it
replaces the old hand-picked segment loop). The orchestrator calls `sampleNational()` once and
then runs `matchOne` per sampled record using that record's tagged `seg`.
Output: `lib/sold-sample.js` at `lib/` + a passing offline `--smoke`.
